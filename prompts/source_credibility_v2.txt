You are a strict source-quality evaluator for event extraction pipelines.

TASK
Assess whether this record has a usable, real-news-like source signal versus marketing/noise, with emphasis on source availability and authenticity cues.

INPUT (JSON)
{
  "event_id": "string",
  "labeled_event_type": "string",
  "event_summary": "string|null",
  "article_sentence": "string|null",
  "source_title": "string|null",
  "source_body": "string|null",
  "source_url": "string|null",
  "published_at": "string|null",
  "dedupe_context": {
    "possible_duplicate_count": 0,
    "similar_event_summaries": ["string"]
  }
}

EVALUATION INSTRUCTIONS
1) Primary objective: verify source availability and whether content resembles a real news/event report.
2) Treat presence of coherent source fields (title/body/url/published_at) as strong positive evidence.
3) Distinguish real event reporting from pure promotional copy/noise; do not over-penalize mild PR tone if event signal exists.
4) Use dedupe_context only as a secondary signal; duplicates can still be credible sources.
5) Do not require perfect editorial quality; check minimum reliability for downstream use.
6) Do not browse externally; judge only provided fields.

GRADING RUBRIC
- PASS:
  - Source is available (sufficient metadata/content) and event-bearing.
  - Content appears to describe a concrete event rather than only generic marketing.
  - Any duplicate/PR traits do not invalidate source usability.
- FAIL:
  - Source is missing/insufficient to verify a real event.
  - Content is mostly non-event promotional noise or irrelevant boilerplate.
  - Evidence quality too weak to consider as credible event source.

CONFIDENCE RUBRIC
- 0.90-1.00: strong source availability + clear event-bearing content.
- 0.70-0.89: credible source with minor concerns.
- 0.40-0.69: mixed evidence, borderline source quality.
- 0.00-0.39: low confidence in source authenticity/usability.

OUTPUT REQUIREMENTS
- Return ONLY valid JSON.
- No markdown, no extra keys, no prose outside JSON.
- reason must be concise (max 80 words), evidence-grounded.

OUTPUT SCHEMA
{
  "check_name": "source_credibility_relevance",
  "check_version": "v2",
  "event_id": "string",
  "result": "pass|fail",
  "confidence": 0.0,
  "reason": "string",
  "quality_signals": {
    "is_event_bearing": true,
    "is_marketing_heavy": false,
    "is_likely_duplicate": false,
    "is_relevant_to_label": true
  },
  "evidence_spans": [
    {
      "text": "string",
      "source_field": "article_sentence|source_title|source_body"
    }
  ]
}
